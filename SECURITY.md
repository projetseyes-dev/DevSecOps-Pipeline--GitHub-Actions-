# Security Policy

## Reporting a Vulnerability

Si vous découvrez une vulnérabilité de sécurité dans ce template ou dans une application qui l'utilise, **ne créez pas d'issue publique**.

### Canal préféré

- Email : `security@org.example` (clé PGP disponible sur le keyserver organisationnel).
- Programme de divulgation responsable : voir page interne *Responsible Disclosure*.

### Informations à fournir

1. Description de la vulnérabilité.
2. Étapes de reproduction (PoC).
3. Impact estimé.
4. Version / commit affecté.
5. Vos coordonnées (optionnel — anonymat respecté si demandé).

### Engagement de l'équipe

| Étape                          | Délai cible                |
|-------------------------------|----------------------------|
| Accusé de réception           | 24 heures ouvrées          |
| Triage initial & sévérité     | 3 jours ouvrés             |
| Plan de remédiation           | 7 jours ouvrés             |
| Correctif Critical            | ≤ 30 jours                 |
| Correctif High                | ≤ 60 jours                 |
| Communication post-mortem     | Après déploiement du fix   |

## Versions supportées

Seule la version la plus récente du template est maintenue. Les correctifs ne sont pas backportés sur les forks.

## Hall of Fame

Les chercheurs ayant signalé responsablement une vulnérabilité sont remerciés (avec leur accord) sur la page interne dédiée.
